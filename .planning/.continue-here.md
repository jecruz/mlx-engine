## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|----------------------|
| Removing materialization barriers without stream-lifecycle proof | Lazy restored arrays can carry cache-I/O-thread stream state into generation. | blocking | Reproduce warm restore, inspect row errors, run quality compare, and keep `mx.eval(...)` unless a stronger replacement proves safe. |
| Promoting faster restore shape without decode stability | Prefix checkpoint restore reduced restore time but failed quality/decode checks. | blocking | Require task-diverse or VLM quality compare to pass before promotion. |
| Mistaking row-level failure for passing benchmark | Harness can write a report even when a prompt row records backend failure. | advisory | Inspect `row_error` and quality compare, not only process exit status. |
